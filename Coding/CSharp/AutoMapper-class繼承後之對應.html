Scenario

All data in one data model

有時候，資料要進行傳遞，可能又要進行呈現

舉例來說，若現在需要呈現User經過隱碼的手機號碼

但是Server後續傳遞資料還是需要完整手機號碼，那就可以規劃如下 Data Model

public class UserInfo
{
    public string Cellphone { get; set; }
    public string MaskedCellphone { get => SomeMaskFunction(this.Cellphone); }
}

Cons

目前 data model 之中只有2個 property

但有些情況下，property有可能會有好幾十個甚至上百個

這時要在一個data model中看出哪些property是顯示用，哪些是資料傳遞用就會很不方便

Split data model

因此一種規畫方式，就是切分 data model

讓資料的傳遞靠 DTO[1]，而資料的呈現靠 View Model

然後用繼承的方式把他們合起來

public class BaseUserInfo
{
    public string Cellphone { get; set; }
}

public class UserInfoVM: BaseUserInfo
{
    public string MaskedCellphone { get => SomeMaskFunction(this.Cellphone); }
}

Cons

而這時新的問題就來了

當從其他來原拿到 BaseUserInfo 後，若要assign進 UserInfoVM

最直觀暴力的做法，就是直接塞

public class UserInfoVM: BaseUserInfo
{
    public string MaskedCellphone { get => SomeMaskFunction(this.Cellphone); }

    public UserInfoVM(BaseUserInfo baseUserInfo){
        this.Cellphone = baseUserInfo.Cellphone;
    }
}

但當 property變多時，這做法就會有很多問題了

  1. 有幾個 property 就塞幾次，很花時間

  2. 若 base class 的 property 一異動，一致性不會獲得保證

    只要忘了將 inherited class constructor 中的 property塞入base的property

    那就會產生邏輯上的bug，而這個是compile階段，compiler檢查不出來的

    要run time才會知道這個錯

Auto Mapper

這時使用auto mapper就很方便了

只需要設定好對應，就可以自動將property去做一一對應了

using AutoMapper;

namespace ConsoleApp
{
    class Program
    {
        static void Main(string[] args)
        {
            // 在初始階段建立對應
            Mapper.Initialize(cfg => 
            {
                // 建立 source(Base) 跟 Target(Test) 的對應
                // source & target 是有方向性的
                cfg.CreateMap<Base, Test>();
            });

            var b = new Base() { AAA = "AAA" };
            var test = new Test(b) { BBB = "BBB" };
            // test.AAA == base.AAA
            ;
        }
    }

    public class Base
    {
        public string AAA { get; set; }
    }

    public class Test: Base
    {
        public string BBB { get; set; }

        public Test(Base b)
        {
            Mapper.Map(b, this);
        }
    }
}

  1. DTO(Data Transform Object)，通常用以傳遞資料用 ↩︎
