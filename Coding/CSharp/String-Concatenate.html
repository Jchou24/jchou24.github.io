字串串接是很常見之應用、像是爬蟲、動態產生網頁內容、產生機器學習資料 ... 等等等

許多場景下都會需要做字串的串接處理

Scenario

一般在做字串處理時，大致上不外乎就是

  1. 根據需求情境計算出一些字串後組合字串

  2. 將大量類似之字串組合成一個字串

String Concatenate

即字串串接，像是

  1. 組 csv 中一筆 row 各 field 之字串

    ... 地址代碼 地址 ...
    ... 123 AAA ...

    像上方表格之地址AAA，可能就是要查另一個表，去看123會轉出什麼地址

  2. 組出 html select 中之單個 option

    <select name="name">
    ...
    <option value="option_value">option_name</option>
    ...
</select>

    其中 option 你可能還會需要看是否

    1. selected?
    2. disabled?
    3. 特別的class
    4. data-attr
    5. ...

Batch String Concatenate

即大量同質性的字串串接

承上方之舉例

  1. 組合 csv 中之各個 row

    ... 地址代碼 地址 ...
    1 ... 123 AAA ...
    2 ... 234 BBB ...

    像上方之表格，可能會需要將 [ "123, 'AAA'", "234, 'BBB'" ] 組合成 "123, 'AAA'\n234, 'BBB'"

  2. 將 html select 中之各 option 組合起來

    像 [ "<option value='0'>A</option>", "<option value='1'>B</option>" ] 組成 "<option value='0'>A</option><option value='1'>B</option>"

Convenient Skill

string format

"+" operator

為最傳統、最直接之字串編輯方式

string a = "Hello";
string b = "World";
string message = a + " " + b

但壞處為，程式可讀性較低

以及大量字串處理時，有可能會有程式執行效率不佳的問題[1]

String.Format

以一種類似填空的方式組字串

ex

string a = "Hello";
string b = "World";
string c = String.Format("{0},{1}",a,b);
// c => "Hello, World"

好處:

  1. 字串模板可以變數化
  2. 程式可讀性佳

壞處:

  1. 不能用名字去對應，只能用順序(0,1,2)去對應

Interpolated Strings

也是以一種類似填空的方式組字串，但使用上更直觀

string a = "Hello";
string b = "World";
string c = $"{a}, {b}";
// c => "Hello, World"

好處:

  1. 更加直觀
  2. 可以用名字去對應

壞處:

  1. 字串模板較難變數化

string builder

在做大量字串串接時，若都是使用 + 去做字串串接

那程式可能會跑得非常沒有效率 [2]

因此針對這種情境，可以使用 string builder

bad efficiency

string [] array = new [] { "1", "2" ... "1000" };
string result = "";
foreach(var item in array){
    result += item + ", ";
}
// result => "1, 2, ... 1000, "

string builder

string [] array = new [] { "1", "2" ... "1000" };
StringBuilder stringBuilder = new StringBuilder();
foreach(var item in array){
    stringBuilder.Append(item + ", ");
}
string result = stringBuilder.ToString();
// result => "1, 2, ... 1000, "

string join

其實 string.Join 跟 string builder 很像

他算是 string builder 的一種使用特例

string [] array = new [] { "1", "2" ... "1000" };
string stringForJoin = ", ";
string result = string.Join(stringForJoin, array);
// result => "1, 2, ... 999, 1000"

但他有個很大的好處是，1000的後面不會出現 ", "

他只會在中間插入 999 個 ", "
