由於最近有朋友想要學習 vue 的東西

這邊就稍微整理一下自己學 vue 的一些心得，並儘可能用比較"平滑"的方式來談談 Vue

本系列文預計將會介紹 Vue 的一些常見用法以及常見語法

Why Vue?

一個網頁會由許多"要素"組成

  • User跟網頁的互動體驗(UX)

  • 網頁上的畫面(UI)

  • 資料流的更新

  • 開發成本(學習難易度、開發時間、重用性、模組化、語法兼容)

  • ...

而使用前端框架可以很大程度的讓上述的事情，在實作上變得有效率

Efficient

這邊簡述使用Vue如何對上述提及議題帶來幫助

網頁動畫呈現

像是切換分頁時，有時淡進淡出的網頁效果會讓使用體驗更好

而這可以透過 vue 的 transition 快速的做到

<transition name="fade">
    <div v-if="isDisplay">
        ...
    </div>
</transition>

像以上寫法，當 isDisplay 被設為 true 時 div 就會套用 fade 的效果淡進

當然 fade 需要撰寫對應的 css ， 但這邊暫且不談的太細節

網頁上的畫面(UI)

網頁上的畫面，由許多的元素組成

而一個元素可以是一個表單，一個按鈕，一個列表

如果畫面要差異更新，那就要使用一大堆的 ajax 去讀資料，並更新畫面

而這些更新的畫面若想要可以分享給別人看到一樣的內容

那就要把當前的狀態與網址做對應的處理(也就是要做front-end routing)

如果想要用網址一一對應的話，那就要在更新資料的同時，手動調整網址

如果用前端框架，那就可以用框架的機制去做 front-end routing

routeConfig = [{
    path: '/',
    name: 'Home',
    component: Home,
}]

而假如 routing 的定義檔這樣寫，那就代表網址對應到 / 時

會幫你帶入到 Home 這個組件，而 Home 這個組件就包含了首頁該有的 html、css、js 了

資料流的更新

事件驅動(event driven)

當 ajax 取得資料後，畫面上必然會需要更新對應的元素

當使用 jquery 或原生的 event listener 進行開發時

整個網頁的開發就會變成 事件驅動(event driven)

當按下某個 button 後，觸發資料的更新，而資料更新後再去更新對應的畫面

資料驅動(data driven)

但使用前端框架進行開發是一種 資料驅動(data driven) 的開發體驗

你會先定義要關注哪些資料，在撰寫設定，當資料更新時

響應式(reactive) ，自動觸發哪些function，自動對有關資料進行調整

並且網頁上的內容，直接響應資料的變動自動更新

所以你不需要再手動更新 html 的內容了，你在開發畫面時

只需要定義好你要依照哪個資料去進行畫面的渲染(render)

資料更新後，框架就會幫你更新畫面(而且框架還會幫你處理更新畫面的效率問題)

<div>{{temperature}}</div>

像以上這個寫法，當系統自己每 30 秒擷取一次溫度資料時

你只需要更新 temperature 這個變數，網頁上的溫度數值又會自己響應資料的變化進行更新

這樣就不會當資料更新時，忘記要更新畫面上的哪些元素了

資料驅動的方向性

但用 data driven 也不是萬能，他算是目前來說比較好的開發模式

但這個開發模式也是需要考慮一些對應的issue

也就是 one-way-data-binding 與 two-way-data-binding

這邊我們暫且先提及到他，先不深入進行討論

開發成本

開發是需要考量成本與效益的，而開發成本需要考量非常多的面向

這邊就先稍微列舉幾個來談談

學習難易度

這最直接關乎到開發人力的量，畢竟市場上工程師就這麼多

技術的學習難易度，直接影響到一個系統有沒有人開發、維護

而 Vue 在學習曲線上就與 Angular、React 有著極大的不同

Vue 是 Progressive Framework (漸進式框架)

你可以用 vue 寫出 js 套件後，用最傳統的方式，在 html 引用套件

也可以用 vue 進行整套的開發

言下之意，你不需要學習 vue 的全部特性後才能用他

你可以只使用他的一小部分功能，就進行開發

然後隨著一步步的學習，在相同專案下，嘗試使用他的不同功能

但 react 跟 angular 就沒辦法了

要使用前，必須先學習相對來說比較多的背景知識後才能使用他們

在前端世界變化極大極快的這個時代，我還滿喜歡這種 Progressive

漸進式的思維，比較不用為了某些新功能，導致必須要一口氣學很多東西後才能使用

開發時間

使用前端框架開發、如果在技術上已經有一定熟練度的話

在開發有規模的系統上時，開發時間將會巨量減少

光是資料更新後，要更新對應畫面這件事，就不是傳統的開發模式能匹敵的

傳統的開發方式，當系統開發到一定程度後，不同功能不同人開發

資料更新後很難去追蹤你要更新哪些畫面，但使用前端框架的話

這些事框架都會規範你寫好定義，並直接用這個定義去運行

直接幫你省下檢查的時間，並增加了畫面更新的可靠性

重用性、模組化

假如今天要製作一個日曆

那麼可能要提供按鈕點擊時切換月份，切換時還要有轉場效果

而日曆的布局可能還需要 css 去撰寫，甚至可能要靠 js 輔助

而在不借助框架下，最原生的開發方式，就是分別寫 html、js、css

然後將他們引用進來

但只要有一個不小心，js沒有閉包，變數命名重複，css作用域重複

開發偵錯時間直接倍量成長，bug變feature

而原生開發模式若要用新的 js 語法模組化元件，封裝元件的話

又不可避免的要"很手動"(自己架設轉譯流程)的處理瀏覽器兼容的問題

使用前端框架的話，像 vue 就有提供 組件(component) 的開發模式

甚至可以使用 SFC(Single File Component) 的方式開發

一個組件，一個檔案，一個模組

模組化的開發方式，直接幫你避掉上述的困難點

用 vue-cli 架設專案的話，還幫你處理了瀏覽器兼容

檔案大小最佳化...的功能

語法兼容

當開發時想使用比較新的 js 語法時，很常要考慮瀏覽器兼容的問題

尤其常常要害怕網站跑在上古神獸 IE 上會掛掉

若專案用 vue-cli 架設，js 常見語法直接幫你編譯成 IE 也懂得語法

許多新規格的語法、語法糖，就可以不用再靠像是 pollyfill 的方式來兼容了

開發思維

以下提供一些開發學習時的思考方向

(以下提到的內容，下一篇文章會有更細節的說明)

  • 在 router 設定哪個網址對應到哪個 View組件

  • 而一個 View組件 再去使用 子組件

  • 這個子組件若很多地方可以用上的話，就可以設計開發的模組化一點，增加共用性

  • 用 SFC 的方式開發的話，一個檔案一個組件，檔案包含 html、js、css，看起來會長這樣

    <template>
    <!-- 這邊寫html -->
</template>

<script>
    export default {
        name: "compoentName",
        components:{ 
            // 註冊要用到哪些組件
        },
        props:{
            // 定義要從外部獲得什麼資料，甚至可以做到資料的驗證
        },
        data(){
            return {
                // 定義組件內的共用資料
            }
        },
        watch:{
            // 定義要監聽哪些資料，讓該資料變化時觸發指定 function
        },
        computed:{
            // 定義組件內的"響應式"衍伸資料
        },
        methods:{
            // 定義共用的function
        },
        mounted(){
            // 定義當組件被掛到 html 上時，要執行哪些動作
        },
        beforeDestroy(){

        }
    }
</script>

<style lang="scss" scoped>

</style>

  • 更多可以定義的地方(hook)，可以參考官網

  • 父組件的資料可以使用html property 的方式傳遞給子組件

    而子組件要在 props 定義對外的 html property

    這樣當外層丟給 props 的資料變化時，子組件就會響應

    <!-- I am parent -->
<children myName="AAA" />

    // I am children
{
    props:{
        myName:{ // 當父組件將 myName 做 assign 時，這邊的值也會響應變化
            type: String // 其他型態還有很多，像是 Number, Objects, Array ...
            required: true // 定義父組件是否一定要傳遞這個值
        }
    }
}

  • 子組件 props 收到的資料不要直接 assign 他，會破壞響應式機制

    要往父組件傳遞的話，可以使用 this.$emit 往父組件傳遞訊息

    假如使用了 this.$emit('CustomEvent')

    父組件可以透過 <ChildrenComponent @CustomEvent="SomeHandler" /> 的方式處理

    這就是2個方向的one-way-data-binding

    而這有語法糖 .sync 可以包裝成 two-way-data-binding

  • 當父組件使用子組件後，想要在子組件內寫html，可以使用 slot

    <father>
    <children>
        <div>I want write something here</div>
    </children>
</father>

    <!-- children.vue -->
<div class="children">
    <slot></slot>
</div>

  • 當資料的傳遞必須透過多層的組件由上傳遞時，可以使用 provide/inject

  • 當資料的傳遞必須透過多層的組件由下往上傳遞時，可以使用 vuex store

  • 組件有許多生命週期，像是建立、掛載到html、從html上卸載、等等等

    你可以在不同的生命週期處理對應的事

    像是在掛載到 html 上之前( Mounted() )前，先準備好資料

    在組件被移除前( beforeDestroyed() )前，把 event listener 清掉

  • 當 css 樣式只希望作用在當前組件，又不想面臨作用域問題的話，可以使用 scoped

    <style scoped>
    // 裏頭的style只會作用在"當前"組件
    // 影響不了外部組件
</style>

  • 若想寫 scss 也是可以的

    <style lang="scss" scoped>
    // ...
</style>
