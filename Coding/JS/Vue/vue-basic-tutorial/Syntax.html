Component 組件

使用vue開發的一大好處，就是可以將網頁上之元素組件化

一個組件便封裝好網頁元素之 html,js,css style

一個vue之組件，會取名叫做 myComponent.vue

一個component打開來後會有類似以下的格式

<!-- 在此放置元素之html -->
<template>
    <div></div>
</template>

<!-- 在此放置元素之js -->
<script>
    export default {
        ...
    }
</script>

<!-- 在此放置元素之css style -->
<style lang="scss" scoped>

</style>

Import 組件

範例位置: http://localhost:8080/ImportComponent

檔案位置: src/views/tutorial/ImportComponent.vue

要使用組件前，需要使用import來引用組件

// @ is an alias to /src
import DemoImport from '@/components/DemoImport.vue'

這代表被引用的組件在 src/components/DemoImport.vue

引用組件組件可以使用絕對路徑、相對路徑

  • 絕對路徑:

    • @/components/DemoImport.vue
    • @代表的是/src

  • 相對路徑:

    • 引用當前目錄組件: ./AAA.vue
    • 引用上層目錄組件: ../BBB.vue

v-model 基礎

範例位置: http://localhost:8080/VModel

檔案位置: src/views/tutorial/VModel.vue

當有時想讓user在網頁上藉由像是input來輸入內容

可以使用v-model來讓輸入內容存到變數中

<input v-model="text" />

而text需要在js中進行宣告

<script>
    export default {
        name: "VModel",
        data(){
            return {
                text: "initial text"
            }
        },
    }
</script>

注意在data中，要用return回傳object，不然會多個組件共享同一個data

Watch

範例位置: http://localhost:8080/Watch

檔案位置: src/views/tutorial/Watch.vue

當需要某變數一被改變就做出對應計算時，可以使用watch監聽該變數

比方說，user在input中輸入英文字時，想要轉大寫

<input v-model="text" />

<script>
    export default {
        name: "Watch",
        data(){
            return {
                text: "initial text",
                upperText: "INITIAL TEXT"
            }
        },
        watch:{
            text(newValue){
                this.upperText = newValue.toUpperCase()
            }
        }
    }
</script>

Computed

範例位置: http://localhost:8080/Computed

檔案位置: src/views/tutorial/Computed.vue

承 Watch 的範例，有些變數想要更快速的計算得到

也就是 某B變數 一改變 某A變數 就跟著計算完成，也可透過 computed 做到

<input v-model="text" />

<script>
    export default {
        name: "Computed",
        data(){
            return {
                text: "initial text",
                // upperText: "INITIAL TEXT"
                // Computed宣告upperText後，data不能宣告一樣的，反之亦然
            }
        },
        computed:{
            upperText:{
                get(){
                    return this.text.toUpperCase()
                }
            }
        },
    }
</script>

跟 watch 的差異

computed一定會得到一個新的reactive演伸變數

但有時可能是某變數一改變就要去更新其他的東西，不單單只是一個變數

那就可以使用watch

Methods

範例位置: http://localhost:8080/Methods

檔案位置: src/views/tutorial/Methods.vue

在組件中，當有共通的methods需要宣告時，可以放在methods裡

承轉大寫的範例，將轉大寫的行為抽到methods裡的toUpper中

<script>
    export default {
        name: "Methods",
        data(){
            return {
                text: "initial text",
                // upperText: "INITIAL TEXT"
                // Computed宣告upperText後，data不能宣告一樣的，反之亦然
            }
        },
        computed:{
            upperText:{
                get(){
                    return this.toUpper(this.text)
                }
            }
        },
        methods:{
            toUpper(string){
                return string.toUpperCase()
            }
        }
    }
</script>

Mounted

範例位置: http://localhost:8080/Mustache

檔案位置: src/views/tutorial/Mustache.vue

有些初始化動作，我們需要在組件掛載到Dom tree上後再進行

像是呼叫 jquery 更改畫面的 UI

那就可以把這些動作放到mounted中

承轉大寫的範例，將字串的初始化動作放到mounted中

<script>
    export default {
        name: "Mounted",
        data(){
            return {
                text: "",
            }
        },
        computed:{
            upperText:{
                get(){
                    return this.toUpper(this.text)
                }
            }
        },
        methods:{
            toUpper(string){
                return string.toUpperCase()
            }
        },
        mounted(){
            this.text = "initial text"
        }
    }
</script>

v-bind

範例位置: http://localhost:8080/VBind

檔案位置: src/views/tutorial/VBind.vue

當需要對html元素之property(像是class, style, href...等等)動態進行修改時，可以使用v-bind

以修改div之class為例

<div class="demo-vbind-class" :class="dynamicClass">
    my color will change at {{timer}} seconds
</div>

:class="dynamicClass"代表，class屬性將隨著dynamicClass這個變數改變

<script>
    // 當組件被掛載，5秒後動態更改組件的class
    // 5秒後，組件之class會多一個'change'，並改變顏色

    export default {
        name: "VBind",
        data(){
            return {
                timer: 5,
                dynamicClass: []
            }
        },
        mounted(){
            let that = this

            let interval = setInterval(function(){
                that.timer -= 1

                if(that.timer <= 0){
                    clearInterval(interval)
                    that.dynamicClass = ["change"]
                }
            }, 1000)
        },
    }
</script>

html元素之property有很多，像是class，style

甚至<a>上常用href，等等

這些property都是可以用v-bind來操作的

另外，針對style，建議為了網頁效能，多利用class屬性來改變元素展示效果會比較好

盡量避免直接操作style

v-on

範例位置: http://localhost:8080/VOn

檔案位置: src/views/tutorial/VOn.vue

當有event需要處理時，在vue上可以有幾種做法，以click為例:

  1. vue原生用法: @click
  2. jquery用法: on click
  3. js原生用法: addEventListener
<div @click="HandleClick"></div>

這代表該div元素被click時，將會觸發HandleClick這個method

詳細範例細節請參考 src/views/tutorial/VOn.vue

備註:

使用 js原生用法: addEventListener 時，要記得在卸載組件前，把listener remove掉

不然會一直留在網頁上變僵屍

v-if

範例位置: http://localhost:8080/VIfElse

檔案位置: src/views/tutorial/VIfElse.vue

當網頁上之元素希望滿足條件後，才掛載到dom上，可以使用v-if

<div v-if="isToggle" class="demo"></div>
<div v-else class="demo change"></div>

v-if="isToggle" 代表當 isToggle 這個變數為 true 時才會掛載該元素

當 isToggle 為 false 時，該元素會從dom上被移除

v-for

範例位置: http://localhost:8080/VFor

檔案位置: src/views/tutorial/VFor.vue

當有元素需要批次產生時，可以使用v-for

<div v-for="(setting, index) in elementSettings"
    :key="index"
    class="demo"
    :class="setting.classSetting"
>{{index}}. {{setting.text}}</div>

data(){
    return {
        elementSettings:[{
                text: "hello",
                classSetting: []
            },{
                text: "hello",
                classSetting: ["change"]
            },{
                text: "world",
                classSetting: []
            }]
    }
},

需要注意的是，:key 建議要給予一個unique的值

Props

範例位置: http://localhost:8080/Props

檔案位置: src/views/tutorial/Props.vue

當需要傳遞資料給組件時，可以透過props來向下，向組件傳

但被傳遞的組件無法透過props中之變數向上傳遞資料

此外，props也可以用來做形別檢查，以及值域檢查

詳見官方說明

<!-- 'src/components/DemoProps.vue' -->
<script>
    export default {
        name: "DemoProps",
        props:{
            text:{
                type: String,
                default: () => "Hello"
            },
            number:{
                type: Number,
                required: true,
                default: () => "Hello"
            }
        },
    }
</script>

這代表DemoProps這個組件，可以接收text，number這兩個自定義屬性，用法如下

<DemoProps :number="1" />
<DemoProps :number="2" :text="text" />

$emit

範例位置: http://localhost:8080/Emit

檔案位置: src/views/tutorial/Emit.vue

當組件要對父組件傳遞資料時可以使用emit，並且使用emit時，可以自定義event

<!-- 'src/components/DemoEmit.vue' -->
<template>
    <div>
        <h3>{{text}}</h3>
        <button @click="HandleClick">click me</button>
    </div>
</template>

<script>
    export default {
        name: "DemoEmit",
        props:{
            text:{
                type: String,
                default: () => "Hello"
            }
        },
        methods:{
            HandleClick(event){
                this.$emit("CustomEvent", event)
            }
        }
    }
</script>

當按下button時，DemoEmit這個組件，會對父組件傳遞一個叫做CustomEvent的自定義event

<!-- src/views/tutorial/Emit.vue -->
<DemoEmit :text="text" @CustomEvent="HandleCustomEvent" />

在父組件中就可以使用@CustomEvent來接收此event，與對應資料

Two-Way Data Binding

範例位置: http://localhost:8080/TwoWayDataBinding

檔案位置: src/views/tutorial/TwoWayDataBinding.vue

當使用<input>時，我們可以使用v-bind來將變數跟元素bind在一起

若自己開發的組件也想使用v-bind，就要用到 Two-Way Data Binding 的技巧

基本想法，就是靠props將資料往子組件送，而當子組件本身資料有異動時

再透過emit將資料往父組件送

詳細語法請參考 src/components/DemoTwoWayDataBinding.vue 與 src/views/tutorial/TwoWayDataBinding.vue 這兩個檔案

Vuex

範例位置: http://localhost:8080/Store

檔案位置: src/views/tutorial/Store.vue

當組件的引用關係變的很深很複雜時，使用vuex來傳遞資料會是比較好的做法

vuex有點像是前端的資料庫，或者全域變數，或者可以說是資料的集散地

以以下情境為例做個說明:

當組件的引用關係如下

* Store.vue
  * ALevelOne.vue
    * ALevelTwo.vue
  * BLevelOne.vue
    * BLevelTwo.vue

若 ALevelTwo.vue 要向 BLevelTwo.vue 傳遞資料

靠props與emit，ALevelTwo.vue必須先往上emit到Store.vue再透過props往下傳到BLevelTwo.vue

而透過使用vuex則可以，ALevelTwo.vue傳給vuex

BLevelTwo.vue使用computed的變數監聽vuex中之變數去自動的拿到資料

vuex在專案中之位置為src/store.js，詳細實做請參考專案

此外，vuex本身還有許多的用法，請參考官網說明

Router

檔案位置: src/router.js

當我們希望某個網址對應到某個組件時，可以透過router來做到

實做請參考專案中之 src/router.js 與 src/views/App.vue

基本運作原則如下

當網址異動，router會根據router.js中之規則，尋找現在要掛載哪個組件

而當找到組件時，變會在 src/views/App.vue 中之 <router-view/> 將該組件掛載上去

詳細用法請參考官網
